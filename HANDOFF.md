# Handoff — tokentally

## Historical context (condensed from sessions 2026-04-22 → 2026-04-30 09:30)

Key architectural decisions and non-obvious findings worth remembering:

### Wails / Windows threading

- Wails must own the main goroutine on Windows (WebView2Loader requirement); systray runs in a goroutine (`getlantern/systray` calls `runtime.LockOSThread()` internally — safe in any goroutine)
- Quit: `os.Exit(0)` directly — `systray.Quit()` deadlocks the Win32 message loop
- `HideWindowOnClose: true` keeps the Wails runtime alive so the tray can re-show the window
- "Open Dashboard" uses a brief `WindowSetAlwaysOnTop(true/false)` pulse to force foreground focus on Windows
- WebView2 native context menu is **disabled** in production Wails builds — must implement custom JS context menus for any right-click UX

### JS binding namespace

- Correct path is `window.go.app.App.*` (lowercase `app` = Go package name); `window.go.App.*` silently hangs
- Use `const App = window.go.app.App` alias in entry point — single fix point if package name changes

### Wails build / frontend

- `wails.json` `frontend:build` uses `--prefix inspector` — Wails runs lifecycle commands from `frontend/`, so `frontend/inspector` would double the path
- `vite.config.ts` must `define: { 'process.env.NODE_ENV': '"production"' }` for IIFE builds — Vue's dev checks reference it and the bundle silently fails without it
- `frontend/web/wailsjs/` is gitignored — regenerated by `wails build` (without `-skipbindings`); `window.go.app.App.*` direct calls work at runtime without stubs

### Session Inspector (Vue SPA at `frontend/inspector/`)

- Progressive render: `visibleCount` starts at 20, RAF-incremented by 20; first paint is immediate
- `batchQueryToolCalls` — single `IN (...)` query replaces N+1 per-turn loop
- Copy buttons on all turn types; HTML export via `app.go SaveHTMLExport` (native Save-As dialog + `os.WriteFile`)
- Export button is disabled (`:disabled="!chunks.length"`) when session has no chunks

### Data Management (Settings)

- Scan Now + configurable retention purge in Settings → Data Management card
- `PurgeMessages` deletes `tool_calls` then `messages` in a transaction but **never touches `files`** — `files` rows are "already scanned" markers; deleting them would cause re-import from disk
- Auto-purge fires after every scan tick regardless of scan errors (independent operations)

### Prompts view

- `<synthetic>` is a literal model name Claude Code writes for subagent sidechain records (`isSidechain: true`)
- Hook records have `type='attachment'` in the DB; included in Prompts when `prompt_text != ''`
- Both show with distinct icons (zap = hook, bot = subagent, person = user) and tag pills in the detail modal
- Skill name for subagent prompts is not available — never written to JSONL

### DB architecture (completed 2026-04-30)

- `db.Pool`: Read handle (`MaxOpenConns=4`) + Write handle (`MaxOpenConns=1`) to same SQLite file — eliminates "database is locked" when UI writes overlap scanner transactions
- `:memory:` paths share one handle so tests work
- FTS5 `messages_fts` with trigram tokenizer — preserves substring-match UX (≥3 chars required per token); triggers keep index in sync; `sanitizeFTSQuery` quotes+ANDs terms
- Schema migrations: `schema_version` row in `plan` table; `targetSchemaVersion = 3`; `readSchemaVersion` infers version from legacy gate flags for existing DBs; append to `migrations` slice + bump constant to add a migration
- For "drop column from existing table": inspect `sqlite_master` first, no-op if already clean (see `migrateDropToolCallsAutoincrement`)
- Indexes: `idx_messages_parent`, `idx_tools_message_uuid`, `idx_tools_use_id`
- DSN: `synchronous=NORMAL` (WAL recommendation, ~2× write speedup); `busy_timeout=5000`
- `Pool.CheckpointWAL()` runs `PRAGMA wal_checkpoint(TRUNCATE)` after each scan tick
- `scanMaps` returns `make([]map[string]any, 0)` (not nil) — Wails serializes nil slice as JSON `null`, which crashes Vue `v-if` guards
